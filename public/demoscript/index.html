Overview

What this demo shows

An anonymous website visitor browses vehicles and chats with the Agentforce vehicle advisor. The moment they share their email, Salesforce Data Cloud resolves their identity in real time — triggering an automated flow that creates an enriched lead and populates a unified 360° profile.

  1. Data Cloud resolves the anonymous visitor to an existing CRM contact
  2. An Automation Event-Triggered Flow fires on the resolved data graph
  3. A Lead is created in Salesforce with enriched vehicle and customer data
  4. A Lead360 component surfaces a unified profile — website, CRM, and Databricks — in one view

Act 1

The Anonymous Visitor

Click Path

  2. Open browser Developer Tools (F12 / Cmd+Opt+I) → Console tab. Find the device ID in the SDK initialization output — look for anonymousId, a 16-character hex string like 6da275aa6822a483. Copy it.
  3. Open the D360 Inspector — click the globe icon (top-right of the site). Paste the device ID into the field and click Lookup. It should show ANONYMOUS.
  4. Click on a vehicle (e.g. Harmonic Sedan)

  6. Browse back, view a second vehicle (e.g. Regulator Truck), configure it too

Act 4

The Automation Event-Triggered Flow

Click Path
  1. In Salesforce Setup, navigate to Flows
  2. Open Real_Time_Lead_Capture
  3. Walk through the flow canvas
What to Explain
  1. Flow type: Automation Event-Triggered — fires on Data Cloud events (data graph changes), not CRM record changes
  2. Trigger: A leadSubmit engagement event appears in the RealTimeLeads Data Graph
  3. Person Account resolution: Checks for an existing account by DC Individual ID → email → creates new if neither matches
  4. Lead de-duplication: Active lead within 60 days? Enrich it. Older or missing? Create a new lead.
  5. Vehicle enrichment: Lead is stamped with model, trim, color, and SKU from the web session
Talking Points
"This flow fires the moment the data graph changes — in this case, when a leadSubmit event hits the resolved identity. It's not polling, it's not scheduled — it's event-driven and real-time."
"The business logic here is just an example. We implemented person account resolution, lead de-duplication with a 60-day window, and vehicle enrichment. But this is fully customizable — your rules would be completely different. Maybe you want to route high-value leads to a specific queue, trigger a follow-up if the customer has an open service case, or notify the nearest dealer. The point is: you have the full power of Salesforce Flow to act on real-time Data Cloud events."
Value Point
Real-time automation at scale. Automation Event-Triggered Flows bring Data Cloud's real-time intelligence directly into Salesforce's automation engine — native, not custom-built. Because it runs on Data Cloud's event architecture, it scales to high volumes without traditional governor limit constraints.

Help

Troubleshooting

Persona not clean before the demo
Open /tools → Account Status → enter the email → Inspect. If NEEDS RESET, use Reset Account → Preview → Confirm Reset.
Device ID not visible in console
Open DevTools (F12 / Cmd+Opt+I) → Console tab. The SDK logs on initialization — look for anonymousId. If the console is clear, reload the page and watch for the first log line.
D360 Inspector shows ANONYMOUS after lead submission
Enable Auto-refresh and wait up to 30 seconds. If still anonymous, check browser console for SDK errors — the identity events may not have fired. Ensure the Agentforce agent captured the email correctly.
Lead wasn't created in Salesforce
Check if the flow is active: Setup › Flows › Real_Time_Lead_Capture. Flow activation can take up to 1 hour after toggling.
Lead360 shows "No Person Account"
The flow creates one — wait a few seconds and refresh. If still missing, check Setup › Automation › Flow Execution Log for errors.
Databricks panels show "No data found"
The MDM ID must resolve. Check the Lead360 header — if MDM ID shows "—", this contact may not have a Party Identification record. Use a persona from the validated list below.
Web Activity panel is empty
Uses email-based device resolution. If contact point email events haven't landed yet, wait 2–3 minutes and refresh.
